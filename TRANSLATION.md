# RET-anchor Translation Boundary

This file defines the translation boundary rules
for **RET-anchor**.

Because RET-anchor is the **public boundary membrane**
of RET, translation errors may directly introduce
authority drift, doctrine drift, or operational reinterpretation.

Translation therefore preserves **boundary semantics first**.

---

## Canonical Source

- Canonical source language: **Japanese**
- All non-Japanese texts are derived translations
- In case of divergence, the Japanese source prevails

Translation does not create new authority.

---

## Translation Invariants

The following semantic contours must remain unchanged.

### 1. Boundary must remain negative

Terms describing exclusion must not drift into:

- guidance
- recommendation
- approval
- usage instruction
- implementation advice

Anchor excludes.
It does not guide.

---

### 2. Public reference must not become legitimacy

Words such as:

- reference
- citation
- adjacency
- membrane
- boundary

must never be translated into terms implying:

- endorsement
- official approval
- certification
- governance legitimacy

---

### 3. Core separation must remain non-commutative

Any wording around RET-core separation must preserve:

- outside relation
- non-translatability inward
- non-reversibility
- no public reinterpretation path

No translation may soften this into
“explains” or “bridges” RET-core.

---

## High-Risk Terms

The following concepts require extra review.

- authority
- legitimacy
- doctrine
- policy
- governance
- membrane
- boundary
- exclusion
- adjacency

These terms are the primary sources of
public overreach drift.

---

## Review Rule

If a translation makes RET-anchor feel like:

- a guide
- a manifesto
- a public doctrine
- a governance layer
- an approval source

the translation is non-compliant.

Reduce explanatory wording.
Restore exclusion semantics.
