# RET-anchor Publication Scope

This file defines the **maximum safe publication boundary**
for RET-anchor.

The purpose is to preserve
public referability without enabling
authority drift or institutional overreach.

---

## Safe for Public Release

The following materials are safe for:

- GitHub public repositories
- Zenodo archival
- citation and preprints
- structural maps
- academic discussion

### Included
- README.md
- INVARIANTS.md
- OVERREACH_PATTERNS.md
- directory structure
- boundary descriptions
- canonical language notice
- license

These materials preserve membrane semantics.

---

## Conditional Release

The following require contextual review:

- translated explanatory notes
- talks or slides
- derivative diagrams
- examples involving public systems
- educational summaries

Release only if they do **not** imply:

- recommended usage
- human behavioral guidance
- organizational adoption
- ethical compliance pathways

---

## Avoid Public Release

The following should remain minimized or private
if they increase authority inheritance risk:

- downstream implementation examples
- system deployment case studies
- institution-facing templates
- policy-adjacent interpretations
- approval-like wording
- normative checklists

These forms increase overreach probability.

---

## Zenodo Boundary

Zenodo publication is acceptable
only as a **structural reference archive**.

The DOI must not be framed as:

- certification
- endorsement
- authority proof
- implementation legitimacy

Archive preserves reference,
not institutional force.

---

## Reduction Rule

If public exposure increases the chance of:

- doctrine formation
- governance reuse
- approval inheritance
- public authority claims

reduce the publication surface immediately.
